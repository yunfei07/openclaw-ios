# OpenClaw iOS Chat UI Design (2026-02-01)

## 目标
- 把聊天界面升级为“微信风”对话 UI（柔和圆润、信息清晰、阅读舒适）。
- 维持现有功能与结构（连接状态、消息列表、输入区、设置页），只改 UI/UX。

## 视觉基调
- 风格：清爽、圆润、低噪声，轻分层。
- 对话形式：左右对齐气泡，双侧尾巴，头像参与排布，留白充足。
- 背景：系统分组背景 + 轻微纵向渐变（白 → secondarySystemBackground）。

## 色彩与字体
- 主色（发送气泡/按钮强调）：柔和绿色（接近微信但更低饱和）。
- 接收气泡：白色或 systemGray6。
- 状态条：成功绿、警告黄、错误红（低饱和）。
- 文字：系统默认主色，保证对比度 ≥ 4.5:1。
- 字体：系统圆角设计 `Font.system(..., design: .rounded)`，无需引入自定义字体。

## 页面结构与组件

### 顶部状态条
- 胶囊提示条，显示连接状态/错误。
- 高度约 28–32，左右内边距 12。
- 状态变化采用 200ms easeInOut。

### 消息列表（聊天区）
- 使用 ScrollView + LazyVStack（避免 List 默认样式）。
- 头像：36x36 圆形裁切，和气泡顶部对齐，间距 8。
- 对方消息：左对齐，气泡最大宽度 72–78%。
- 自己消息：右对齐，文本左对齐阅读更舒适。
- 气泡：圆角 16–18，内边距 12–14，阴影更弱或无，左右尾巴增强“对话感”。
- 空状态：柔和文案 + 简短引导，不要强视觉元素。

### 输入区
- 胶囊输入框 + 右侧圆形发送按钮。
- 输入框支持 1–4 行；按钮禁用状态置灰。
- 触控目标 ≥ 44x44。

### 设置页（Gateway/Token）
- Form 替换为卡片化分组。
- 主按钮“保存并返回”填满宽度；危险按钮“清除 Token”用描边红。
- 可选：Token 眼睛图标显示/隐藏。

## 动效
- 新消息进入：`move(edge: .bottom)` + `opacity`（180–220ms）。
- 流式回复：仅微弱淡入，不做大幅动画避免抖动。

## 可访问性
- 文字对比度 ≥ 4.5:1。
- 动态字体支持。
- 触控目标 ≥ 44x44。
- 错误信息用清晰颜色与简短文案。

## 非目标
- 多会话列表
- 复杂时间轴
- 自定义字体引入
- 深色模式专属主题（仍需保持系统适配）

## 验收标准
- 聊天页面具备微信式气泡与对齐。
- 输入区为圆角胶囊 + 圆形发送按钮。
- 设置页为卡片化布局。
- 状态条与错误提示更清晰且不抢占空间。
